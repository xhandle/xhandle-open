# Code of Conduct

## Our Commitment

We want xHandle to be a welcoming, respectful, and constructive open-source project.

Contributors, maintainers, and community members should help create an environment where people can ask questions, share ideas, give feedback, and collaborate safely regardless of experience level, background, identity, or personal characteristics.

## Expected Behavior

- Be respectful and professional.
- Assume good intent, and give feedback constructively.
- Focus on the work, not the person.
- Be welcoming to new contributors and basic questions.
- Accept that people may have different technical backgrounds, priorities, and communication styles.

## Unacceptable Behavior

- Harassment, intimidation, or discrimination.
- Personal attacks, insults, or derogatory language.
- Publishing private information without permission.
- Deliberate disruption, trolling, or bad-faith engagement.
- Any conduct that would be inappropriate in a professional collaborative environment.

## Scope

This Code of Conduct applies in project spaces, including GitHub issues, pull requests, discussions, and other communication channels directly associated with xHandle.

## Reporting

If you experience or observe behavior that violates this Code of Conduct, contact the maintainers at:

- `nick.malloy@interlocksystems.io`

Please include relevant context, links, screenshots, or timestamps if available.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, or other contributions that violate this Code of Conduct. Maintainers may also temporarily or permanently restrict participation for serious or repeated violations.
