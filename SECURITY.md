# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in xHandle, please do not open a public GitHub issue.

Instead, report it privately by emailing:

- `nick.malloy@interlocksystems.io`

Please include:

- a clear description of the issue
- steps to reproduce it
- affected files, routes, or features if known
- any suggested remediation or mitigation

## Response Expectations

We will make a good-faith effort to:

- acknowledge receipt of the report
- investigate the issue
- determine an appropriate remediation path

Response times are best-effort and may vary depending on severity and maintainer availability.

## Scope Notes

xHandle is a local-first project, but security issues may still exist in areas such as:

- backend API routes
- secret handling
- authentication or license flows
- local storage and IndexedDB data handling
- dependency or deployment configuration

If you are unsure whether something is security-relevant, it is still helpful to report it privately.
